# Obsidian 零基础实操教程

> 不堆概念，直接上手。看完这篇，你就能用 Obsidian 写笔记了。

---

## 一、这是什么？为什么要用？

**一句话**：Obsidian = 一个在电脑上写笔记的软件，但它有一个杀手锏——**笔记和笔记之间可以互相链接**，像大脑里的神经元一样，最终形成一张你自己的知识网络。

**核心优势**：

| 优势 | 什么意思 |
|------|---------|
| 📁 **本地文件** | 你的笔记就是电脑上的普通 `.md` 文件，不绑定任何平台，永远不会丢 |
| 🔗 **双向链接** | 笔记 A 链到笔记 B，B 会自动显示"被 A 引用了"——你能看到知识之间的关联 |
| 🧠 **知识图谱** | 所有笔记之间的链接关系，可视化成一个网络图，直观看到你的知识结构 |
| 🆓 **完全免费** | 个人用不要钱，所有核心功能免费 |
| 🔌 **插件生态** | 1000+ 社区插件，按需扩展功能 |

3. 安装 → 打开

## 三、创建你的第一个「仓库」（Vault）

> **仓库（Vault）** = 一个文件夹，里面装你所有的笔记。

打开 Obsidian 后会看到这个界面，点击 **Create a new vault**：

```
┌──────────────────────────────────┐
│         ⚡ Obsidian              │
│                                  │
│  [ Create a new vault ]  ← 点这  │
│                                  │
│  [ Open folder as vault ]        │
│  [ Open vault from Obsidian Sync]│
└──────────────────────────────────┘
```

然后：
1. **Vault name**：给你的知识库起个名字，比如 `我的知识库`
2. **Location**：选一个文件夹位置，建议放 D 盘（不影响 C 盘空间）
3. 点 **Create**

完成后你会看到 Obsidian 的主界面：

```
┌─────────┬──────────────────┬─────────┐
│  左侧   │                  │  右侧   │
│  文件   │                  │  可以   │
│  列表   │   笔记编辑区      │  打开   │
│         │                  │  反向   │
│  📁文件  │   在这里写笔记    │  链接   │
│  📁文件  │                  │  面板   │
│         │                  │         │
└─────────┴──────────────────┴─────────┘
```

## 五、核心必杀技：双向链接 🔗

**这是 Obsidian 最强大的功能，没有之一。**

### 怎么用？超级简单

在任意笔记里，输入 `[[` 两个左方括号，然后输入另一个笔记的名字，Obsidian 会自动弹出匹配列表：

```
输入：[[双
      ┌──────────────────┐
      │ [[双向链接]]      │  ← 选这个
      │ [[双向奔赴]]      │
      └──────────────────┘
```

选中后就变成 `[[双向链接]]`，这是一个**可点击的链接**。按住 Ctrl 点它，就跳到那个笔记。

如果链接的笔记**还不存在**呢？没关系！`[[还没写的笔记]]` 照样可以写，点击它 Obsidian 会**自动帮你创建**那个笔记。

### 反向链接（Backlinks）

这是双向链接的另一半。打开任意笔记，看右侧面板（如果没有就点右侧的 🔗 图标）：

```
┌─ 反向链接 ──────────────┐
│ 链接到当前文件           │
│                         │
│ 📄 今日学习：Obsidian入门 │
│   提到了 [[当前笔记]]     │
│                         │
│ 📄 周报_06-08          │
│   提到了 [[当前笔记]]     │
└─────────────────────────┘
```

你不需要手动追踪"谁引用了我"——Obsidian 自动帮你追踪。

### 链接的四种写法

| 写法 | 效果 |
|------|------|
| `[[面试记录]]` | 链接到「面试记录」笔记 |
| `[[面试记录\|查看面试]]` | 链接但显示为"查看面试" |
| `[[面试记录#候选人评价]]` | 链接到「面试记录」里的「候选人评价」标题 |
| `[[面试记录#^abc123]]` | 链接到笔记里的某个段落 |

## 六、组织笔记：三个工具配合用

很多人一开始就纠结"该用文件夹还是标签还是链接"——答案是**三个都用**，但各有分工：

### 🗂️ 文件夹 = 物理存放位置

用来做**大分类**，不要搞太深（最多 2-3 层）。

推荐结构（PARA 方法）：

```
我的知识库/
├── 00-收件箱/       ← 临时笔记、灵感速记，定期清理
├── 01-项目/         ← 当前在做的事（招聘季、培训项目）
├── 02-领域/         ← 长期关注的领域（劳动法、绩效管理）
├── 03-资源/         ← 参考资料、收藏的文章
├── 04-归档/         ← 已完成、不再活跃的内容
└── Templates/       ← 笔记模板
```

### 🏷️ 标签 = 跨分类的属性标记

用来标注笔记的**状态、类型、优先级**：

```markdown
#待处理 #重要 #HR/招聘 #HR/培训
```

- `#待处理` — 状态标签
- `#重要` — 优先级标签
- `#HR/招聘` — 嵌套标签，表示归属

**建议**：一开始只建 5-10 个核心标签，不要贪多。

### 🔗 链接 = 笔记之间的关联

这是最灵活的。不要纠结"该放哪个文件夹"——放一个地方，然后用链接从其他地方引用它。

```markdown
# 某候选人面试记录

候选人：张三
应聘岗位：[[前端开发工程师]]
面试轮次：二面
关联职位：[[2026春招-前端JD]]
面试题库参考：[[面试题库#前端]]
```

### 三者的分工

```
文件夹 → "这东西大概属于哪一类？"
标签   → "这东西是什么状态/类型？"
链接   → "这东西跟什么有关？"
```

## 七、图谱视图：看见你的知识网络

按 **Ctrl+G**（Mac: Cmd+G）打开图谱。

```
          ● ← 一个笔记
         /|\
        ● ● ● ← 被链接的笔记
       /  |  \
      ●   ●   ●
```

- 点越大 = 被链接次数越多 = 越重要
- 孤立的点 = 还没跟其他笔记建立联系
- 连接线 = `[[链接]]` 关系

**每周看一次图谱**：发现孤立笔记，想想它应该跟什么关联。

## 八、必装的 5 个插件

**原则**：痛点驱动，不要一次性装几十个。以下 5 个起步足够：

### 1. Templater（模板引擎）

**解决什么问题**：每次创建面试记录、周报等重复性笔记，不用从头写。

**怎么用**：
1. 设置 → 第三方插件 → 关闭安全模式 → 浏览 → 搜 "Templater" → 安装并启用
2. 在 `Templates/` 文件夹创建一个模板文件，比如 `面试记录模板.md`：

```markdown
---
date: <% tp.date.now("YYYY-MM-DD") %>
type: 面试记录
tags: [HR/招聘, 面试]
---

# {{候选人姓名}} - 面试记录

## 基本信息
- 应聘岗位：[[ ]]
- 面试轮次：
- 面试官：

## 面试评价

### 优势

### 不足

### 总体评价

## 下一步
- [ ] 发送反馈
- [ ] 安排下一轮
```

3. 新建笔记时，按 **Ctrl+Shift+T**（或 Mac: Cmd+Shift+T），选模板插入。

### 2. Calendar（日历视图）

**解决什么问题**：按日期查看每日笔记，一目了然。

**怎么用**：
1. 先启用核心插件 **Daily notes**（设置 → 核心插件 → Daily notes → 开启）
2. 安装社区插件 **Calendar**
3. 右侧面板会出现日历，点任意日期直接创建/打开那天的日记
4. 有日记的日期会显示一个点

### 3. Dataview（数据查询）

**解决什么问题**：把笔记当成数据库来查询。比如"列出所有待处理的候选人"、"统计本月面试数量"。

**先装好就行**，具体用法见下一节。

### 4. Editing Toolbar（编辑工具栏）

**解决什么问题**：不习惯 Markdown 语法？这个插件给你一个像 Word 一样的格式工具栏。

**怎么用**：安装启用后，编辑区顶部出现一行工具栏：
```
[B] [I] [U] [H1] [H2] [H3] [🔗] [📷] [☑] [表格] ...
```
点按钮就能加格式，不用记语法。

### 5. Easy Typing（中文优化）

**解决什么问题**：中文写作时自动在中文和英文/数字之间加空格，自动转换标点。

安装即用，无需配置。

## 九、Dataview 入门：把笔记变数据库

> Dataview 是 Obsidian 最强大的插件之一。先了解概念，用到的时候再深挖。

### 核心概念

每篇笔记可以在**开头**加一段 YAML（也叫 Frontmatter），用来存结构化数据：

```markdown
---
title: 张三面试记录
date: 2026-06-08
type: 面试记录
candidate: 张三
position: 前端开发工程师
round: 二面
status: 通过
tags: [HR/招聘, 面试]
---
```

然后在另一个笔记里，用 Dataview 查询：

### 列出所有待跟进候选人

```dataview
TABLE candidate AS "候选人", position AS "岗位", round AS "轮次"
FROM "01-项目/招聘"
WHERE status = "待跟进"
SORT date DESC
```

### 统计本月面试数量

```dataview
TABLE date AS "日期", candidate AS "候选人", status AS "结果"
FROM "01-项目/招聘"
WHERE date >= date("2026-06-01")
SORT date DESC
```

### 列出所有带 #重要 标签的笔记

```dataview
LIST
FROM #重要
SORT file.name ASC
```

### 显示所有未完成任务

```dataview
TASK
FROM "01-项目"
WHERE !completed
```

> 💡 **先用纯文本记，等你发现有查询需求了再回头配 Dataview。** 不要一开始就搞复杂。

## 十、日常工作流（以 HR 为例）

### 场景 1：面试记录

1. 按 Ctrl+N 新建笔记
2. 用 Templater 插入面试模板
3. 填写候选人信息，`[[链接]]` 到岗位 JD 和面试题库
4. 添加 `#HR/招聘` 标签
5. 面试后在同一笔记里追加评价

### 场景 2：每周周报

1. Calendar 插件点周一日期 → 创建日记
2. 用 `[[链接]]` 关联本周处理的面试、培训、项目
3. Dataview 自动汇总本周未完成任务

### 场景 3：培训材料整理

```markdown
# 新员工入职培训

## 培训模块
1. [[公司文化与制度]]
2. [[劳动合同与社保]]
3. [[办公工具指南]]

## 培训记录
| 日期 | 人数 | 讲师 | 反馈 |
|------|------|------|------|
| [[2026-06-01]] | 12 | 王老师 | 满意 |
| [[2026-06-15]] | 8 | 李老师 | 待收集 |
```

### 场景 4：劳动法知识库


## 十一、常见问题

### Q：Obsidian 怎么同步到手机？

| 方案 | 费用 | 难度 | 推荐 |
|------|------|------|------|
| **Obsidian Sync**（官方） | $4/月 | ⭐ 最简单 | 最省心 |
| **iCloud** | 免费（Mac+iOS） | ⭐⭐ | 苹果用户首选 |
| **Remotely Save 插件** + OneDrive/WebDAV | 免费 | ⭐⭐⭐ | 跨平台免费方案 |
| **Obsidian Git 插件** + GitHub | 免费 | ⭐⭐⭐ | 有版本控制需求 |

### Q：笔记多了会不会很乱？

不会。这正是双向链接的威力——你可以用 `[[链接]]` 从任意角度组织同一篇笔记，不需要把它放到唯一的文件夹里。

### Q：需要每天写日记吗？

建议从周记起步，每周花 10 分钟回顾。习惯后再过渡到日记。

### Q：图片和附件怎么处理？

把图片直接拖进 Obsidian 编辑区就行。建议在设置里指定一个 `Attachments/` 文件夹统一存放。
