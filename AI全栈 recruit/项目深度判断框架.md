## 四个层次

### L1：API 调用层（Demo 级）
只会调 API，拼 prompt，套壳。

**简历信号：**
- 「基于 ChatGPT/DeepSeek 实现 XX 对话」
- 「调用大模型 API」
- 「Prompt 优化」

**判断：** 淘汰。这不是"做 AI"，这是"用 AI 产品"。

### L2：组件集成层（能用级）
能搭架子，调框架默认参数，能跑但没调优。

**简历信号：**
- 「用 LangChain/LlamaIndex 搭建 RAG」
- 「用 SpringAI 集成 LLM」
- 技术栈罗列全但 **没有效果指标**

**判断：** 待定。看其他维度是否突出。

### L3：系统设计层（生产级）
有架构决策、有调优过程、有量化结果。

**简历信号：**
- 「设计了 XX 架构」「自研/自建」
- 「chunk size 从 X 调到 Y，TopK 从 A 换到 B」
- 「混合检索」「重排序」「HyDE」「Self-RAG」
- **有准确率数字**（且会说从多少提到多少）

**判断：** 通过。项目深度达标。

### L4：架构创新层（主导级）
不是用别人的框架拼，而是自己设计了组件间的协作范式。

**简历信号：**
- 「设计多智能体协作架构」「A2A 通信协议」「球权状态机」
- 「自研 Agent 框架」
- 「Plan-Execute 模式」「三层 Agent 架构」
- 能说清楚 Agent 之间怎么通信、谁管调度、异常怎么回滚

**判断：** 强烈推荐。这是真正主导过 Agent 架构的人。

## 简历信号速查

| 浅层信号（L1-L2） | 深层信号（L3-L4） |
|---|---|
| 「使用 XX 框架」 | 「设计/自研 XX 架构」 |
| 「实现了对话功能」 | 「准确率从 X% 提到 Y%」 |
| 「集成了 LLM」 | 「做参数调优，最终确定 chunk=512, TopK=5」 |
| 「调用 API」 | 「自建通信协议/状态机」 |
| 罗列一堆技术栈 | 关键点有量化数据 |
| 多人协作的"参与" | 「独立负责/独立开发」 |

## 关键判断原则

1. **同一个项目名不等于同深度。** 简历里写同一个项目（如 Clowder AI），有人写「构建多角色协作」，有人写「NDJSON适配层+A2A球权状态机」——前者是概念，后者是实现路径。
2. **能说清"怎么做的"而不是"做了什么"的，就是深度。**
3. **有量化结果的优先级高于罗列技术栈的。**
4. **面试验证点：** 如果信号模糊（用词像 L3 但没有具体实现细节），面试时需要追问：「你的 chunk 分割策略具体怎么选的？有没有遇到检索失败的 case？怎么修的？」
